# 时空知识图谱页面 (Spatio-Temporal Knowledge Graph)

## 页面路径
- 主路径: `/data-factory/stkg`
- 可从 Data Factory 页面的"深入了解｜时空知识图谱"按钮跳转

## 页面结构

### Section A - 从地图到图谱：空间智能的必然演进
- 展示传统地图到知识图谱的演化过程
- 强调 AI 时代空间智能的系统性升级
- 包含线框风格的演进示意图

### Section B - 什么是时空知识图谱（工程视角）
- 工程定义：实体优先、时间一等公民、关系可计算、持续演化
- 展示核心对象示例（地址、建筑、AOI、道路等）
- 说明实体的三大属性：唯一空间身份、生命周期、多源属性绑定

### Section C - 技术路线：如何构建可生长的时空图谱
- 四步流程展示：
  1. 多源时空数据接入
  2. 空间实体建模
  3. 时序关系构建
  4. 持续更新与演化
- 每步包含关键挑战说明

### Section D - 核心技术能力拆解
- 4 张能力卡片：
  1. 空间实体统一标识体系
  2. 时空一致性约束机制
  3. 自动化建图与更新能力
  4. 图查询与推理能力

### Section E - 地图×图谱交互（核心功能）⭐
这是页面的核心交互部分，包含：

#### 交互功能
1. **地图视图** (左侧)
   - 展示 AOI、地址点、道路等空间实体
   - 点击实体触发图谱联动
   - 根据状态变化显示不同颜色

2. **图谱视图** (右侧)
   - 力导向布局展示知识图谱
   - 节点类型：空间实体、服务类型、商品品类、事件
   - 点击节点触发地图联动

3. **时间轴控制** (底部)
   - 三个时间点：09:00（正常）、10:30（繁忙）、11:05（异常）
   - 拖动时间轴观察状态演化
   - 显示当前事件和状态

4. **控制面板** (顶部)
   - 实体类型筛选：全部/AOI/地址/道路
   - 关系层级：1-hop / 2-hop
   - 事件筛选：全部/业务事件/异常

#### 示例数据
已植入完整的物流场景示例数据：

- **空间实体**：物流园区、收货点、道路
- **服务类型**：小件快递、快运、大件、冷运、同城急送
- **商品品类**：生鲜水果、医药制品、大型设备
- **事件**：冷运事件、运单激增、拥堵
- **关系**：位于、支持、承载、影响、限制等

### Section F - 面向机器与 Agent 的世界模型底座
- 强调机器需要的三大能力：
  1. 可理解的空间语义
  2. 可调用的世界状态
  3. 可推理的行动后果

### Section G - 与传统方案的关键差异
- 对比表格展示 6 个维度的差异：
  - 核心组织方式
  - 时间处理
  - 数据更新
  - 面向对象
  - 决策能力
  - 关系表达

### Section H - 开放与共建 (CTA)
- 强调共建价值
- 两个 CTA 按钮：联系合作、返回核心技术

## 技术特点

### 视觉风格
- 沿用 Data Factory 页面风格
- 紫色渐变主题 (#9c27b0)
- 深色背景 (#0f0f1e → #1a1a2e → #1d1a2e)

- 地图与图谱双向联动
- 时间轴状态演化
- 实时高亮与过滤
- 响应式布局

## 访问方式

1. 直接访问: `http://localhost:5173/data-factory/stkg`
2. 从 Data Factory 页面点击"深入了解｜时空知识图谱"按钮

## 特色功能

### 可演示的交互
- ✅ 点击地图实体，图谱自动聚焦
- ✅ 点击图谱事件，地图高亮相关实体
- ✅ 拖动时间轴，观察状态演化
- ✅ 三种状态变化：正常 → 繁忙 → 异常

### 数据完整性
- ✅ 5 种物流服务类型
- ✅ 3 种商品品类
- ✅ 3 个时间点事件
- ✅ 多种关系类型（空间、能力、承载、事件、约束）

### 工程可信度
- 语气克制、专业
- 强调技术判断
- 避免营销化表述
- 突出工程逻辑

## 注意事项

- 示例数据为演示用途，实际部署时需要接入真实数据源
- 地图视图使用简化展示，可根据需要集成真实地图服务
- 图谱布局使用简化算法，大规模数据需要优化性能
