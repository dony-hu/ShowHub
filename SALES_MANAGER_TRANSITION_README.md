# 销售经理能力跃迁 - 培训网页

## 项目概述

这是一个面向销售培训的临时网页，名称为**销售经理能力跃迁**。该网页展示了销售经理从"拼体力"到"拼脑力"再到"拼算力与心力"的三阶段进化过程。

## 访问地址

**本地开发环境：** `http://localhost:5175/sales-manager-transition`

**导航菜单：** 在顶部导航栏中可以找到"销售能力跃迁"的链接

## 页面结构

### 第一部分：销售能力和发展图谱

- **核心理念介绍**：展示销售经理成长的进化过程
- **三个阶段卡片**（支持左右滑动）：
  - 🏃 **跑动型销售** - "铁脚板与勤务兵"（入门/成长期）
    - 核心定义：执行力至上
    - AI基础使用：用AI写日报、查资料
  
  - 🎬 **导演型销售** - "多面手与组局者"（骨干/主力期）
    - 核心定义：资源整合力
    - AI效率倍增：竞争分析、方案美化、模拟谈判
  
  - 👑 **经营型专家** - "运筹帷幄的操盘手"（专家/管理期）
    - 核心定义：战略经营力
    - AI决策辅助：数据分析、宏观趋势洞察

- **五大核心能力图谱**：
  1. 客户关系 (Customer Insight)
  2. 项目运作 (Project Orchestration)
  3. 产品、工具与数智化 (Product, Tools & AI)
  4. 组织运作 (Organizational Synergy)
  5. 经营与战略 (Business Strategy)

### 第二部分：客户关系 (Customer Insight)

三个并排卡片展示三个阶段的客户关系管理能力：

#### 跑动型销售 - "铁脚板与勤务兵"
- **维度1**：中基层覆盖（执行者）
- **维度2**：中高层触达（破冰者）
- **维度3**：伙伴关系（协作者）
- **维度4**：AI能力（信息采集员）

#### 导演型销售 - "多面手与组局者"
- **维度1**：中高层突破（攻坚者）
- **维度2**：全景洞察（策划者）
- **维度3**：伙伴深度协同（导演者）
- **维度4**：AI能力（情感维护师）

#### 经营型专家 - "运筹帷幄的操盘手"
- **维度1**：战略绑定（顾问者）
- **维度2**：生态统治力（操盘者）
- **维度3**：伙伴驾驭（领袖者）
- **维度4**：AI辅助（趋势预言家）

### 第三部分：项目运作 (Project Orchestration)

展示三个阶段在项目运作中的能力差异：

#### 跑动型销售
- 项目前：嗅觉（感知）
- 项目中：执行（合规）
- 项目后：复盘（成长）
- AI辅助：效率工具人

#### 导演型销售
- 项目前：策划（布局）
- 项目中：导演（整合）
- 项目后：铺垫（延续）
- AI辅助：模拟陪练

#### 经营型专家
- 项目前：锁定（控盘）
- 项目中：收割（获益）
- 项目后：经营（ROI）
- AI辅助：数据决策

### 第四部分：产品、工具与数智化 (Product, Tools & AI)

展示三个阶段对产品、工具和AI的应用能力：

- 流程工具：准确维护与信号弹（合规员）

- 资源应用：策划与口碑建立（导演者）
- AI辅助：内容生成师

## 设计特色

### 色彩系统（深蓝主题）
- **主色调**：深蓝色 (#1a3a52, #2a5282)
- **强调色**：浅蓝/天蓝 (#4a90e2, #5b9fd6)
- **背景色**：浅蓝灰 (#f5f8fb)
- **文本色**：深灰 (#1a1a1a)

### 交互功能
1. **卡片滑动**：第一部分的三个阶段卡片支持左右滑动，并有指示器点击导航
2. **响应式设计**：完全响应式，支持桌面、平板和手机设备
3. **悬停效果**：卡片和链接具有悬停过渡效果，提升用户体验

### 信息组织
- 分层级的标题和小标题清晰度
- 多维度的能力展示（维度框、阶段框、AI能力框）
- 核心心态、能力画像、具体能力要求的递进式展开

## 技术栈

- **框架**：React 18 + TypeScript
- **样式**：原生 CSS（使用 CSS Variables 和 Grid/Flexbox）
- **路由**：React Router v6
- **构建工具**：Vite

## 响应式断点

- **桌面**（>1024px）：完整布局，三列网格
- **平板**（768px - 1024px）：自适应调整
- **手机**（<768px）：单列布局，优化触摸体验
- **小屏**（<480px）：极致简化

## 使用说明

1. 确保开发服务器运行：`npm run dev`
2. 在浏览器中打开：`http://localhost:5175/sales-manager-transition`
3. 或者从主导航菜单中点击"销售能力跃迁"

## 内容编辑指南

### 修改阶段信息

编辑 `SalesManagerTransitionPage.tsx` 中的 `stages` 数组：

```typescript
const stages: Stage[] = [
  {
    id: 'runner',
    name: '跑动型销售',
    subtitle: '"铁脚板与勤务兵"',
    icon: '🏃',
    description: '...',
    period: '入门/成长期'
  },
  // ...
];
```

### 修改核心能力

编辑 `SalesManagerTransitionPage.tsx` 中的 `coreCapabilities` 数组：

```typescript
const coreCapabilities: CoreCapability[] = [
  {
    id: 1,
    title: '客户关系 (Customer Insight)',
    definition: '...',
    elements: ['...']
  },
  // ...
];
```

### 修改样式

编辑 `SalesManagerTransitionPage.css` 文件中的任何样式规则。特别是：

- 修改色彩：编辑 `:root` 中的 CSS 变量
- 修改布局：编辑 Grid 和 Flexbox 相关规则
- 修改字体：编辑 `font-family` 和字号相关规则

## 浏览器兼容性

- Chrome/Edge：完全支持
- Firefox：完全支持
- Safari：完全支持
- IE 11：不支持（使用现代特性如 CSS Grid）

## 性能优化

- 使用 React.lazy 实现代码分割
- 使用 CSS 变量减少样式重复
- 使用 Flexbox/Grid 实现高效布局
- 优化动画使用 `transform` 和 `opacity`

## 故障排除

### 页面不显示
1. 检查控制台是否有错误
2. 确保开发服务器运行正常
3. 清除浏览器缓存（Ctrl+Shift+Delete）

### 样式不正确
1. 确保 CSS 文件正确导入
2. 检查浏览器开发者工具中的计算样式
3. 验证 CSS 变量是否正确定义

### 卡片滑动不工作
1. 检查状态管理是否正确
2. 验证按钮点击事件是否绑定
3. 检查浏览器是否禁用了 JavaScript
